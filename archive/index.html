OmniLedger demo archive

This demo allows you to run a blockchain on your local computer and play around with it. It is based on OmniLedger, which is a blockchain service built upon the ByzCoin paper from the DEDIS lab at EPFL. It has been supported and extended by the C4DT and is now available as a standalone, archived demo that can be run on your computer.

You can find more information on our blog post here: OmniLedger.

When you typed docker compose up, four nodes started communicating with each other and awaiting commands from a client. The first thing you should do is to create a new user connected to the blockchain. To do this, click on the following link: . Now you are ready to look at the different services offered by this demo.

Services

OmniLedger service

Login service and handling users

This service allows administration of a user credential on the blockchain. You can read more in the blog post about Account Management in Omniledger. Internally we used it to connect to other services, like our matrix instance for internal discussions: CAS Login using OmniLedger. Connecting mobile devices was a bit more complicated, but today's passwordless authentication services also involve scanning of QRCodes, like did our solution to Login to Matrix on Mobile. The final service we implemented was a Signup and Recovery tool for OmniLedger. This allowed anybody from EPFL to get an OmniLedger credential and then use it to sign on to our matrix chat.

OL Explorer

Blockchain explorer for programmers

This explorer allows detailed views of the different parts of the ByzCoin blockchain. You can print all fields of the stored blocks and use the console to work on the different fields of the blocks.

Columbus Explorer

Blockchain explorer for humans

This explorer has a nice GUI to search for transactions on the blockchain. You can follow forward- and backward-links to different blocks and display their content. It interprets most of the transactions correctly.

Troubleshooting

If you're stuck and things don't work anymore, it's best to start from scratch. You need the following to do so:

  1. On the login page, you need to delete all local storage
  2. Stop docker compose: docker compose down
  3. Delete the data from the nodes: rm -rf nodes
  4. Start the nodes again: docker compose up
  5. Re-create the login with the link on this page
